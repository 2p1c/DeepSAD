# 多特征融合与多通道融合改进指南

本文档用于指导后续在当前 Deep SVDD 框架上进行模型增强，目标是在不破坏小样本可训练性的前提下，提高对损伤（脱粘、散射）敏感的特征表达能力。

---

## 1. 目标与原则

目标：

- 提升损伤检出能力（提高 AUC、提升健康/损伤分数分离度）
- 保持训练稳定，不引入过度复杂结构
- 让改动可以分阶段验证，便于回滚与对比

原则：

1. 先做“可解释的增量改进”，再做复杂结构
2. 每次只增加一个维度（特征或通道），避免耦合混乱
3. 所有对比都固定同一数据划分与评估脚本

---

## 2. 多特征融合（能量、相位、模态）思路

### 2.1 特征定义建议

在每个窗口内提取以下特征分支（可逐步启用）：

1. 能量特征（Energy）
- 时域能量：`E_time = sum(x^2)`
- 包络能量：`E_env = sum(env(x)^2)`
- 分频带能量：对频段积分（用于模态敏感）

2. 相位特征（Phase）
- Hilbert 瞬时相位：`phi = angle(hilbert(x))`
- 相位导数/瞬时频率：`dphi/dt`
- 与健康参考的相位差统计（可选）

3. 模态相关特征（Mode-aware）
- 频带能量比（低频/高频）
- 主频峰值与带宽
- 关键时延窗口内的局部能量比

建议：先做“低成本统计特征”（标量向量）作为辅分支，再考虑复杂可学习分支。

### 2.2 融合方式（推荐顺序）

#### 方案 A：后融合（Late Fusion，优先）

- 主干 CNN 输出 `z_cnn`
- 手工统计特征 MLP 输出 `z_feat`
- 拼接后投影：`z = Linear([z_cnn ; z_feat])`

优点：
- 实现最简单，风险低
- 手工特征可解释性强

#### 方案 B：中间融合（Mid Fusion）

- CNN 中间层特征与频带/相位分支特征在中层拼接
- 再经卷积或注意力模块融合

优点：
- 交互更充分
风险：
- 参数上升快，小样本更易过拟合

#### 方案 C：门控融合（Gated Fusion）

- 学习门控权重 `alpha_i`，自动调节不同特征贡献
- 形式：`z = sum(alpha_i * z_i)`，`sum(alpha_i)=1`

优点：
- 能适应不同工况下特征重要性变化

建议阶段：A -> C -> B

## 3. 多通道融合（时域、频域、时频域）思路

### 3.1 输入通道组织

可将一个窗口组织为多通道张量：`X in R^(C x L)`

建议通道配置：

- 通道 1：时域原始信号 `x(t)`
- 通道 2：频域幅值 `|FFT(x)|`（可裁剪到目标频段后重采样到长度 L）
- 通道 3：时频特征（如 STFT 能量沿频率或时间聚合后的 1D 序列）

推荐从 `C=2`（时域+频域）起步，再扩展到 `C=3`。

### 3.2 网络结构选择

#### 结构 1：早期通道融合（Early Fusion）

- 直接把 `C` 通道输入同一个 1D CNN（`in_channels=C`）

优点：
- 代码改动小
缺点：
- 不同域特征尺度差异可能影响训练

#### 结构 2：双分支/三分支编码器（推荐）

- 时域分支 encoder_t
- 频域分支 encoder_f
- 时频分支 encoder_tf（可选）
- 各分支输出后拼接融合：`z = Fuse([z_t, z_f, z_tf])`

优点：
- 每个域独立建模，更稳健

#### 结构 3：跨通道注意力融合（进阶）

- 在分支输出后加入轻量注意力（SE 或 MHA）
- 自适应选择最有用通道

建议阶段：2 -> 1（快速对照）-> 3

---

## 4. 与 Deep SVDD 结合方式

无论采用多特征或多通道，最终都要输出统一 embedding `z`，并继续使用：

- 中心初始化：健康训练集前向均值
- 目标函数：`min ||z - c||^2`

关键约束：

1. 不用损伤数据参与训练
2. 融合后 `embedding_dim` 不宜过大（建议 64 或 128）
3. 监控 `embedding_std`，避免 collapse

---

## 5. 分阶段落地路线（推荐）

### Phase 1（低风险）

- 保留现有时域主干
- 加入统计特征分支（能量+相位+频带比）
- 采用 Late Fusion

验收：
- 相比 baseline，`window_auc` 或 `path_auc` 有稳定提升

### Phase 2（中风险）

- 增加频域通道（时域+频域双分支）
- 融合层改为门控融合

验收：
- 损伤分数均值进一步上升
- 健康误报不明显增大

### Phase 3（进阶）

- 引入时频分支（STFT/CWT 派生 1D 描述）
- 尝试轻量注意力融合

验收：
- 在跨批次/跨工况测试上仍有提升

---

## 6. 训练与正则化建议

1. 学习率与权重衰减
- 起点：`lr=1e-3`, `weight_decay=1e-5`
- 分支增多后可降到 `lr=5e-4`

2. 模型容量控制
- 小样本下优先减小隐藏通道而非盲目加深
- 建议总参数量增长不超过 baseline 的 2~3 倍

3. 归一化一致性
- 多通道输入必须分别标准化，避免量纲不一致
- 训练/推理必须使用同一预处理管线

4. 早停与回滚
- 记录每次改动对应 `metrics.json`
- 若 2~3 次实验无提升，回退到上个稳定版本

---

## 7. 评估与对比规范

每次改进至少输出并对比：

- `window_level.auc`
- `path_level.auc`
- `healthy_score_mean` vs `damaged_score_mean`
- 分布图/ROC/PCA

建议建立对比表：

- baseline（仅时域）
- +多特征融合
- +多通道融合（2通道）
- +多通道融合（3通道）

只在同一数据划分、同一随机种子设置下比较。

---

## 8. 最小可行改进（优先实现）

如果时间有限，先做这一版：

1. 主干保持不变（时域 CNN）
2. 额外提取 6~12 维统计特征（能量、包络能量、频带比、相位方差等）
3. 用一个小 MLP 编码统计特征
4. 与主干 embedding 做 Late Fusion 输出最终 `z`

该方案实现成本低、可解释性高，通常是从 baseline 到融合模型的最稳过渡。
